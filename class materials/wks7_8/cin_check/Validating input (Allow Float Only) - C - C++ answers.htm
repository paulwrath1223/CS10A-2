re: Validating input (Allow Float Only)

When the user tries to enter a char value into a float value (or any incompatible type mismatch like this), cin enters the fail state. This indicates that something went wrong, and it can't be trusted for input. When this happens, you need to clear it (to get rid of the fail state indicator) and then flush it (to get rid of whatever caused the error in the first place). This works out to:

Expand|Select|Wrap|Line Numbers
  1. if (!cin) { // This only executes if cin is not working - that is, if cin is in the fail state
  2.    cin.clear(); // Clears the fail state indicator
  3.    cin.ignore(100); // Ignores 100 characters in the input stream.  This is usually more than enough to get rid of the error-causing character.
  4. }
When validating input, obviously you want to keep trying until nothing wrong happens, and you have the proper value in your variable. This indicates that you want a loop; you can combine the above if statement with a while() loop to constantly ask the user for a price until the user actually enters a number rather than a character.

re: Validating input (Allow Float Only)

1) Don't use scanf() since this is C++ and not C.
2) When you cin>>price, the code won't compile. When you ask for help, the code should art least compile. This can't compile because the name of an array is the address of element 0. Therefore, price is a float* and not a float. The compiler will not allow you to change the address of the array.
3) Anytime you use cin>> the assumption is that you know the input format. If you cin>>aFloat and enter a char, a fail bit is set. You have to clear the fail bit and flush your inout buffer
4) Anytime you use cin>> you should check that cin.fail() is false.
